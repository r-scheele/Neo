# Reference Collection Plan

This plan defines what to collect before detailed UI design. It does not require copying any app. The goal is to learn from relevant interaction patterns and synthesize a Neo-specific design system.

## Reference Targets

Collect 8 to 12 references across these categories:

- WhatsApp-first communication
- Mobile commerce operations
- Payment and receipt review
- Helpdesk/inbox triage
- Compact SaaS companion apps
- Calm premium Nigerian or African SME brand systems
- Empty, error, and permission states

## Screens To Capture

- Onboarding setup checklist
- Mobile dashboard or command center
- Inbox list
- Chat detail
- AI or suggested reply card
- Order detail
- Payment/receipt review
- Follow-up or task queue
- Customer profile
- Settings and permissions
- Empty state
- Error/offline state

## Design Synthesis Questions

- Which references help users scan urgent work fastest?
- Which payment states feel trustworthy without feeling scary?
- Which inbox patterns make labels and ownership clear?
- Which screens handle dense data without becoming a compressed desktop dashboard?
- Which illustration styles feel premium, local-aware, and practical without stereotypes?
- Which AI surfaces preserve human control instead of implying automation without review?

## Patterns To Adapt

- A compact daily operations feed instead of a generic dashboard grid.
- Chat rows with business labels and next action.
- AI draft cards that feel reviewable, not magically final.
- Receipt review as a deliberate financial decision screen.
- Status badges with text and icon, not color-only signals.
- Setup checklist that unlocks value gradually.

## Patterns To Avoid

- Full desktop tables on mobile.
- Marketing hero layouts inside the app.
- Neon AI gradients, sparkles, robots, and holograms.
- Generic CRM layouts that ignore WhatsApp selling.
- Payment confirmations that look too easy or automatic.
- Decorative cultural motifs that do not support the task.

## Collection Output

When references are collected later, record each one with:

- Full-screen screenshot or link.
- Screen/category.
- What to keep.
- What to avoid.
- Which Neo screen it informs.
- Any risk if copied too closely.
