Create a high-fidelity mobile app UI design for the Welcome screen.

## 1. Screen Name

Welcome.

## 2. Screen Purpose

Introduce Neo to a new Nigerian WhatsApp-first business owner and invite them to start setup.

## 3. User Goal

Understand that Neo helps organize WhatsApp selling work and tap the main action to begin setup.

## 4. App Context

Neo is an AI-powered WhatsApp commerce operating system for Nigerian SMEs that helps sellers reply customers, capture orders, review receipt screenshots safely, manage follow-ups, and stay in control from one mobile-first command center.

## 5. Target User

Nigerian SME owners and trusted staff who sell through WhatsApp chats, product screenshots, bank transfers, delivery coordination, and polite local customer language.

## 6. Visual Direction

Use the attached master UI style reference if provided. Match Neo's premium, calm, wise, trustworthy, warmly practical style with warm ivory background, deep forest green actions, soft white surfaces, sand borders, muted gold accents, charcoal text, and rounded outline icons.

## 7. UI Style Rules

Use a mobile-first layout, readable humanist typography, 8px radius buttons/cards, subtle border-first surfaces, and no flashy AI styling. Keep the screen welcoming but not like a marketing landing page.

## 8. Layout Structure

Top safe-area brand mark and small status line, centered hero illustration area, concise title/copy block, primary setup button near thumb reach, secondary sign-in link below.

## 9. Header Area

Show the Neo logo mark or a simple text brand lockup, with a compact trust cue such as "For WhatsApp sellers" or "Commerce assistant". Do not use a large nav bar.

## 10. Main Content Area

Feature the approved onboarding hero style: organized chats, receipts, order cards, and follow-up cues in a calm commerce composition. Keep enough whitespace for a premium mobile feel.

## 11. Cards / Lists / Forms / Primary Content Sections

Use one quiet value summary block with 2-3 short rows such as "Reply faster", "Review receipts safely", and "Follow up without forgetting". Each row may use a rounded outline icon.

## 12. Primary Action

A deep forest green full-width button labeled "Start setup" or similar.

## 13. Secondary Actions

A secondary text action for "I already have an account" or "Sign in", visually lower priority.

## 14. Navigation Behavior

This is a logged-out entry screen. Primary action leads to setup/auth. Secondary action leads to Register / Sign In. No bottom tabs.

## 15. Empty / Loading / Error State Notes

No data states are needed on the reference image. If showing a loading state, use a calm inline spinner on the primary button only.

## 16. Asset Usage Instructions

Use or reserve space for `assets/images/onboarding-hero-neo-operating-system.png` and `assets/images/logo-mark-neo.png`. Do not invent unrelated mascots, stock photos, fake WhatsApp logos, or decorative marketplace scenes.

## 17. Text / Copy Guidance

Use clear readable UI labels where possible. Avoid tiny paragraphs. Use realistic short text blocks and labels. Exact final copy will be implemented in code, so prioritize layout accuracy, hierarchy, and component structure over perfect text rendering.

## 18. Mobile Dimensions Guidance

Single full-screen mobile app UI mockup, portrait aspect ratio around 390x844 or 430x932, no phone frame, no browser frame, no desktop layout.

## 19. What To Avoid

Avoid marketing-site hero composition, random gradients, robot imagery, sparkly AI visuals, tiny unreadable feature text, crowded cards, and anything that feels like a generic chatbot app.

## 20. Output Requirements

Create one clean production-ready mobile app screenshot-style mockup with high visual clarity, consistent Neo palette, consistent typography hierarchy, and no external device frame.
