## Explicit Safe Local-Only Copy Found

The code correctly warns users in many places that the current action is local-only. Examples include:

- "No WhatsApp message was sent" remains only for local fixture/demo conversation and follow-up flows.
- Backend receipt decisions now save through the API for durable IDs; local receipt fixtures still warn that no payment verification or message is sent.
- "Decision saved to local UI only" remains only for fixture approval records; durable approval IDs save through the backend.
- "No bank lookup or payment provider was contacted" in backend receipt review decisions, with local mock copy isolated to fixture paths.
- Local permission preview copy remains only for demo/visual gates; backend denied-write responses now route to the permission UI.

These warnings should remain until the real integration works end to end. After integration, they should either be removed or isolated behind explicit dev/demo mode.
