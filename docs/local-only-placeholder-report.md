## Explicit Safe Local-Only Copy Found

The code correctly warns users in many places that the current action is local-only. Examples include:

- "No WhatsApp message was sent" in conversation and follow-up flows.
- "No payment, message, or backend action will run" in receipt review.
- "Decision saved to local UI only" in approvals.
- "No bank lookup, payment provider, or backend was contacted" in receipt review.
- "Production permissions must be enforced by a trusted backend" in permission screens.

These warnings should remain until the real integration works end to end. After integration, they should either be removed or isolated behind explicit dev/demo mode.
