# Error State Prompts

Error states should be calm, recoverable, and non-blaming.

## Offline

```md
Create an error-state illustration for offline mode in Neo.

App context:
Neo is a mobile-first AI-powered WhatsApp commerce operating system for Nigerian SMEs.

Error context:
The user sees this when the app cannot reach the network. The app should help them understand that cached information may be visible but sending replies, confirming payments, and saving settings require connection.

Style reference:
Use the approved Neo style anchor for palette, lighting, soft geometry, and premium editorial flat style.

Concept:
A calm phone-shaped panel with a softened broken-connection line, a small chat bubble, and a warm retry loop motif. No readable UI.

Requirements:
- Transparent background.
- No text, letters, or numbers.
- Helpful, not alarming.
- Use error color subtly.
- Clear at mobile size.

Avoid:
- No scary warning triangle as the main visual.
- No broken-device cliché if it feels harsh.
- No robot.
- No fake controls.

Output:
Save as error-offline.png.
```
