# Empty State Prompt

Use when a screen has no user data yet.

```md
Create an empty-state illustration for [SCREEN_NAME] in [APP_NAME].

App context:
[APP_NAME] is a [TYPE_OF_APP] for [TARGET_USER] that helps them [CORE_OUTCOME].

Empty state context:
The user sees this when [WHY_THE_SCREEN_IS_EMPTY]. The desired next action is [PRIMARY_ACTION].

Style reference:
Use the approved style anchor. If using a mascot, keep the same mascot identity, proportions, color palette, face style, and accessory details.

Concept:
[VISUAL_CONCEPT_THAT_FEELS_ENCOURAGING_NOT_SCOLDING]

Requirements:
- Calm and inviting.
- No text.
- Transparent background if possible.
- Clear at small mobile sizes.
- Leaves visual room for headline, body copy, and button in the app UI.
- Uses the approved palette and line weight.
- Does not make the user feel like they failed.

Avoid:
- No sad or shame-based imagery.
- No complex scenes.
- No random objects outside the visual system.
- No UI screenshots or fake text.

Output:
High-resolution image named `[empty-state-file-name].png`.

Acceptance check:
The image should make the empty state feel intentional and guide the user toward the first useful action.
```
