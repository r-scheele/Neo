# Success State Prompt

Use for completion, confirmation, or celebration moments.

```md
Create a success-state illustration for [SUCCESS_EVENT] in [APP_NAME].

App context:
[APP_NAME] is a [TYPE_OF_APP] for [TARGET_USER] that helps them [CORE_OUTCOME].

Success context:
The user sees this after [ACTION_COMPLETED]. The desired feeling is [FEELING].

Style reference:
Use the approved style anchor. If using a mascot, keep the exact same mascot identity, proportions, colors, face style, and accessory details. Change only the pose and expression.

Concept:
[VISUAL_CONCEPT]

Requirements:
- Positive but not visually noisy.
- No text.
- Transparent background if possible.
- Works inside a mobile confirmation screen or modal.
- Uses approved success color sparingly.
- Clear focal point.
- Feels consistent with the rest of the asset set.

Avoid:
- No excessive confetti unless approved.
- No fake app UI.
- No unreadable symbols.
- No inconsistent mascot or character details.

Output:
High-resolution image named `[success-state-file-name].png`.

Acceptance check:
The image should reward the user without overwhelming the next action.
```
