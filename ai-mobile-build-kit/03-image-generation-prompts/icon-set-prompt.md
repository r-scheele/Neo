# Icon Set Prompt

Use for custom category icons or branded icons. For common UI actions, prefer a proven icon library such as Lucide where appropriate.

```md
Create a consistent icon set for [APP_NAME].

App context:
[APP_NAME] is a [TYPE_OF_APP] for [TARGET_USER] that helps them [CORE_OUTCOME].

Style reference:
Use the approved style anchor for palette, corner style, stroke weight, and personality.

Icons needed:
- `[ICON_1_NAME]`: `[MEANING]`
- `[ICON_2_NAME]`: `[MEANING]`
- `[ICON_3_NAME]`: `[MEANING]`
- `[ICON_4_NAME]`: `[MEANING]`
- `[ICON_5_NAME]`: `[MEANING]`

Requirements:
- Consistent stroke weight.
- Consistent corner radius.
- Same visual size and padding.
- No text.
- Simple shapes readable at 24px.
- Transparent background.
- Works on light background.
- Avoids duplicating standard system navigation icons unless needed.

Avoid:
- No mixed filled and outline styles unless specified.
- No complex illustrations.
- No gradients unless the style guide requires them.
- No brand logos.

Output:
One high-resolution contact sheet for review plus individual transparent PNG or SVG-style exports if supported.

Acceptance check:
The icons should look like one family and remain recognizable at small mobile UI sizes.
```
