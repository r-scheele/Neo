# Onboarding Hero Prompt

Use for the first onboarding screen or welcome screen.

```md
Create an onboarding hero illustration for [APP_NAME].

App context:
[APP_NAME] is a [TYPE_OF_APP] for [TARGET_USER] that helps them [CORE_OUTCOME].

Style reference:
Use the approved style anchor and approved mascot/logo if provided. Preserve palette, proportions, lighting, and illustration style.

Scene:
[DESCRIBE_THE_CORE_OUTCOME_VISUALLY_WITHOUT_TEXT]

Requirements:
- Mobile portrait friendly.
- Clear focal point.
- No embedded text.
- Space at top or bottom for app UI copy if needed.
- Works on [LIGHT/DARK/BRANDED] background.
- Shows the app's emotional promise.
- Avoids showing fake UI details unless requested.
- Composition should remain readable when cropped for different devices.

Avoid:
- No clutter.
- No tiny unreadable interface elements.
- No screenshots of competitor apps.
- No random characters inconsistent with the mascot.
- No text, numbers, or charts unless explicitly requested.

Output:
High-resolution portrait or square image, transparent background if possible.

Acceptance check:
The image should make the product promise understandable in one glance without needing text inside the image.
```
