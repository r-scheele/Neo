# Style Anchor Prompt

Use this first. Every later asset should reference the approved style anchor.

```md
Create the primary visual style anchor for a mobile app.

App context:
[APP_NAME] is a [TYPE_OF_APP] for [TARGET_USER] that helps them [CORE_OUTCOME].

Visual direction:
[MOOD, COLOR_PALETTE, TYPOGRAPHY_FEEL, CARD_STYLE, ICON_STYLE, ILLUSTRATION_STYLE]

Create:
[MASCOT / HERO_ILLUSTRATION / LOGO_MARK / PRIMARY_CHARACTER / ABSTRACT_VISUAL_SYSTEM]

Requirements:
- Mobile app friendly.
- Clean composition.
- No text or letters.
- Works on a light background.
- Consistent color palette.
- High quality.
- Reusable as reference for future assets.
- Clear silhouette at small mobile sizes.
- Balanced spacing with no important detail near the edges.

Avoid:
[WHAT_TO_AVOID]
- No watermarks.
- No mock UI unless requested.
- No brand names.
- No tiny unreadable details.

Output:
Square image, high resolution, transparent background if possible.

Acceptance check:
The image should define palette, proportions, line weight, lighting, and overall mood clearly enough that future assets can match it.
```
