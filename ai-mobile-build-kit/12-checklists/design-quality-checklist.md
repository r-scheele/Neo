# Design Quality Checklist

Use this before implementing or approving screens.

## Direction

- [ ] Visual direction exists.
- [ ] Target user is reflected in the design.
- [ ] Mood is defined.
- [ ] Color palette is defined.
- [ ] Typography feel is defined.
- [ ] Icon style is defined.
- [ ] Illustration style is defined.
- [ ] Motion direction is defined.

## Mobile UI

- [ ] Primary action is obvious.
- [ ] Screen hierarchy is clear.
- [ ] Text is readable.
- [ ] Touch targets are at least 44px.
- [ ] Layout works on small screens.
- [ ] Long content is handled.
- [ ] Safe areas are respected.
- [ ] Keyboard does not block forms.

## States

- [ ] Loading state designed.
- [ ] Empty state designed.
- [ ] Error state designed.
- [ ] Success state designed.
- [ ] Offline state designed where relevant.
- [ ] Permission denied state designed where relevant.
- [ ] First-time and returning states are considered.

## Consistency

- [ ] Colors match the palette.
- [ ] Cards use consistent radius and borders.
- [ ] Buttons use consistent styles.
- [ ] Icons look like one family.
- [ ] Illustrations match the style anchor.
- [ ] No screen introduces a random new visual language.

## Done Looks Like

Design is ready when a screen can be implemented without inventing style decisions during coding.
