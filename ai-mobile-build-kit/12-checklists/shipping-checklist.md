# Shipping Checklist

Use this before releasing to testers or stores.

## Build

- [ ] Version number correct.
- [ ] Build number correct.
- [ ] App icon final.
- [ ] Splash final.
- [ ] Production environment configured.
- [ ] EAS build succeeds.
- [ ] Installable build tested.

## App Quality

- [ ] Core flow works.
- [ ] Auth works if included.
- [ ] Persistence works after restart.
- [ ] Offline behavior acceptable.
- [ ] Loading, empty, error, and permission states work.
- [ ] No dev-only screens visible.
- [ ] No noisy console logs.

## Devices

- [ ] iOS tested if in scope.
- [ ] Android tested if in scope.
- [ ] Small screen tested.
- [ ] Large screen tested.
- [ ] Real device tested.

## Store

- [ ] Screenshots ready.
- [ ] Description ready.
- [ ] Privacy disclosures ready.
- [ ] Support URL ready.
- [ ] Data safety or privacy forms match app behavior.

## Analytics And Privacy

- [ ] Analytics events verified.
- [ ] Sensitive data excluded.
- [ ] Privacy policy aligned.
- [ ] Permissions justified.

## Final Gate

- [ ] Known risks documented.
- [ ] Release blocker list empty.
- [ ] Product owner approved.

## Done Looks Like

The app is ready to ship when the release build has been tested and the store listing truthfully matches the app.
