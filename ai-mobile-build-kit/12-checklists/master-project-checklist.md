# Master Project Checklist

Use this from idea to release.

## Product

- [ ] One-sentence app description written.
- [ ] Target user defined.
- [ ] Main pain defined.
- [ ] Current workaround documented.
- [ ] Core outcome defined.
- [ ] PRD completed.
- [ ] MVP scope and non-goals defined.
- [ ] Feature backlog created.
- [ ] User flows mapped.
- [ ] Screen map created.
- [ ] Data entities listed.

## Design

- [ ] Design references collected.
- [ ] Competitors reviewed.
- [ ] Visual direction written.
- [ ] UI style guide drafted.
- [ ] Screen state inventory completed.
- [ ] Style anchor generated.
- [ ] Asset inventory started.
- [ ] Required assets approved.

## Architecture

- [ ] Stack decision documented.
- [ ] Dependency rules accepted.
- [ ] Architecture plan written.
- [ ] Folder structure defined.
- [ ] State plan written.
- [ ] Persistence plan written.
- [ ] Auth plan written if relevant.
- [ ] Analytics plan written if relevant.
- [ ] Security and secrets guide reviewed.

## Setup

- [ ] Expo app created.
- [ ] Git initialized.
- [ ] `.env` and `.env.example` handled safely.
- [ ] Expo Router configured.
- [ ] NativeWind configured.
- [ ] Asset folder created.
- [ ] Lint command exists.
- [ ] Typecheck command exists.
- [ ] GitHub repository created.
- [ ] Root `AGENTS.md` written.

## Feature Loop

- [ ] Next feature selected.
- [ ] Feature spec written.
- [ ] One prompt selected.
- [ ] Diff reviewed.
- [ ] App run.
- [ ] New feature tested.
- [ ] Related old features tested.
- [ ] Lint run.
- [ ] Typecheck run.
- [ ] Feature committed.

## Release

- [ ] Real device testing complete.
- [ ] iOS and Android edge cases checked where relevant.
- [ ] Production build tested.
- [ ] EAS checklist complete.
- [ ] Store checklist complete.
- [ ] Analytics verified.
- [ ] Privacy and security checked.
- [ ] Final release readiness completed.

## Done Looks Like

The project is under control when every active phase has a checklist and nothing important lives only in chat history.
