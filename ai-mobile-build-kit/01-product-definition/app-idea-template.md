# App Idea Template

Use this to turn a loose idea into a clear starting point. Keep it short enough to revisit before every major decision.

## One-Sentence App Description

`[APP_NAME] is a [TYPE_OF_APP] for [TARGET_USER] that helps them [CORE_OUTCOME].`

Example:

`Tiny Wins is a habit tracking app for busy professionals that helps them build consistent routines through small daily actions.`

## App Snapshot

- App name: `[APP_NAME]`
- Type of app: `[consumer / marketplace / SaaS companion / content / AI tool / utility / other]`
- Target user: `[SPECIFIC_USER]`
- Main pain: `[PAIN]`
- Current workaround: `[HOW_USERS_SOLVE_IT_NOW]`
- Core outcome: `[WHAT_SUCCESS_LOOKS_LIKE_FOR_USER]`
- Primary platform: `[iOS / Android / both]`
- Monetization assumption: `[free / subscription / one-time purchase / marketplace fee / enterprise / unknown]`

## Why Now?

- What changed in the market, user behavior, technology, or your own access that makes this app worth building now?
- What makes a mobile app the right surface instead of a website, spreadsheet, or manual service?

## MVP Boundary

In the first usable version, the user must be able to:

- `[CORE_ACTION_1]`
- `[CORE_ACTION_2]`
- `[CORE_ACTION_3]`

The MVP will not include:

- `[NON_GOAL_1]`
- `[NON_GOAL_2]`
- `[NON_GOAL_3]`

## Done Looks Like

This idea is defined when another person can explain:

- Who the app is for.
- What problem it solves.
- What the first version does.
- What the first version does not do.
