# Feature Backlog Template

Use this backlog to queue features without letting them blur together.

## Backlog Rules

- One row per feature.
- Each feature should be independently buildable.
- Do not add implementation details until the feature is selected.
- Split anything that cannot be reviewed in one diff.
- Every selected feature gets a feature spec before coding.

## Prioritization Questions

- Does this feature support the MVP goal?
- Can it be built and tested independently?
- Does it require new data, auth, permissions, or backend work?
- Can it be simplified?
- What existing feature could it break?

## Done Looks Like

The backlog is useful when the next feature can be selected without rethinking the whole product.
